## 部署前准备

### 1. 安装 Docker

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装依赖
sudo apt install -y ca-certificates curl gnupg lsb-release

# 添加 Docker GPG 密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 添加 Docker 仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 验证安装
sudo docker --version
```

### 2. 创建部署目录

```bash
# 创建项目目录
sudo mkdir -p /opt/simple-web
sudo chown -R $USER:$USER /opt/simple-web
```

## GitHub Secrets 配置

在 GitHub 仓库中配置以下 Secrets：

### SSH_HOST
- **值**：阿里云 ECS 的公网 IP 地址（如 47.92.xxx.xxx）

### SSH_USERNAME
- **值**：SSH 登录用户名（建议使用 root 或具有 sudo 权限的用户）

### SSH_PRIVATE_KEY
- **值**：完整的 SSH 私钥内容
- **生成方式**：
  ```bash
  # 本地生成密钥对
  ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
  
  # 将公钥复制到 ECS
  ssh-copy-id -i ~/.ssh/id_rsa.pub username@ECS_IP
  
  # 查看私钥内容并复制到 GitHub
  cat ~/.ssh/id_rsa
  ```

## 部署流程

### 手动部署（测试用）

```bash
# 1. 登录 ECS
ssh username@ECS_IP

# 2. 进入部署目录
cd /opt/simple-web

# 3. 克隆代码（或上传 docker-compose.prod.yml 和镜像）
# 如果使用 GitHub Actions 自动部署，跳过此步

# 4. 启动服务
docker compose up -d

# 5. 查看状态
docker compose ps

# 6. 查看日志
docker compose logs -f

# 7. 健康检查
curl http://localhost:80
```

### 自动部署（GitHub Actions）

当代码推送到 main 分支时，GitHub Actions 会自动：
1. 运行测试
2. 构建 Docker 镜像
3. 将镜像打包为 tar.gz
4. 传输到 ECS
5. 加载镜像并启动容器
6. 执行健康检查

## 常见问题排查

### 1. SSH 连接失败

```bash
# 检查 SSH 服务状态
sudo systemctl status ssh

# 检查安全组规则
# 登录阿里云控制台 → ECS → 安全组 → 入方向规则
```

### 2. Docker 镜像拉取失败

```bash
# 如果在国内服务器无法访问 Docker Hub，可以配置镜像加速
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
EOF

sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 3. 端口被占用

```bash
# 查看端口占用
sudo lsof -i :80

# 停止占用端口的进程
sudo fuser -k 80/tcp

# 或修改 docker-compose.prod.yml 中的端口映射
```

### 4. 查看容器日志

```bash
# 查看所有容器日志
docker compose logs

# 查看指定容器日志
docker compose logs web

# 实时查看日志
docker compose logs -f web
```

## 回滚操作

如果新版本出现问题，需要回滚到旧版本：

```bash
# SSH 登录到 ECS
ssh username@ECS_IP

# 进入部署目录
cd /opt/simple-web

# 查看可用的 Docker 镜像
docker images | grep simple-web

# 停止当前容器
docker compose down

# 使用指定版本的镜像启动
docker tag <old-image-id> simple-web:latest
docker compose up -d

# 或者从镜像仓库拉取旧版本
docker pull <registry>/simple-web:<old-tag>
docker compose up -d
```
